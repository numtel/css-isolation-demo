CSS Isolation in Meteor Components Comparison

# Option Advantages Disadvantages
1 X-Tags WebComponent
  • Use Meteor template, reactively
  • Light footprint: 17kb gzipped
  • LESS must be compiled separately
2 iframe
  • Use Meteor template, reactively
  • Completely cross-browser friendly
  • LESS must be compiled separately
3 Polymer WebComponent
  • Existing Meteor package: ecwyne:polymer
  • Doesn't use Meteor template
  • Data passed on element attributes (as JSON), or as XML-esque child elements (eek)
  • Largest dependency: 70kb gzipped
  • LESS must be compiled separately

X-Tags WebComponent

  • Any events would need to be specifically coded on the xtag definition. (Or a translation layer...)
{{> xtagExample }}

iframe

  • Styles are fed through LESS rendering on the server. Probably best to go with pre-compiled LESS on production package.
{{> iframeTemplate src="hello" less=less}}

Polymer WebComponent

  • Last test duration reactively set to 'Click Me' counter.
{{> polymerExample }}
